# 📚 도서 URL 자동완성 웹앱 (완전체 v4)

## v4에서 교보문고 5,000원 오탐을 해결했어요
문제: 교보문고는 페이지에 '배송비/적립금/쿠폰' 같은 값이 많아서, 텍스트에서 처음 잡히는 `5,000원`이 판매가로 잘못 들어가는 경우가 있었습니다.

해결:
- 교보문고는 Next.js 기반이라 HTML 안에 `__NEXT_DATA__` JSON이 들어있습니다.
- v4는 가격을 **__NEXT_DATA__에서 우선 추출**해서 오탐을 크게 줄였습니다.
- 텍스트 기반 가격 스캔도 **배송비/적립금/포인트/쿠폰** 근처 값은 제외하도록 개선했습니다.
- 그래도 비정상적으로 낮은 가격(<=6000)이 나오면 Playwright로 1회 재시도합니다.

## 엑셀 출력(요청 반영 유지)
- 처리상태 컬럼 제거
- 상품 URL 컬럼을 맨 오른쪽으로 이동
